# Ruleset v3.3.0 (冻结规范)

本文件是工程侧的“唯一事实来源”(single source of truth)。当原始设计文档存在空缺/冲突时，以本文件为准。

## 版本信息

- `ruleset_version`: `v3.3.0`
- 适用内容目录：`content/v3.3.0/`

## 核心属性

六维属性：

- `stamina` 体能
- `skill` 技巧
- `mind` 心态
- `academics` 学业
- `social` 人际
- `finance` 理财(资源规划)

## 初始分配

- 总点数：250
- 单属性范围：25..60 (含)
- 分配完成后按人格判定逻辑得到 `personality_start`

人格判定优先级与条件来源于 `content/v3.3.0/personality.json`。

补充：若不满足任何非默认人格条件，则判定为默认人格(白纸型)。

## 终局人格 (personality_end)

当 run 结束(包括中途崩解或完成决赛)时，计算 `personality_end`：

- 使用与初始相同的人格判定器，对“终局属性”重新判定。
- 用途：成长报告展示；成就《蜕变的证明》(最终人格 ≠ 初始人格)的判断依据。

## 12周事件流程

- 共 12 周，每周有 6 个固定选项。
- 每周展示：从 6 个固定选项中**确定性**抽取 3 个展示给玩家。
- 玩家选择其中 1 个，立即结算属性变化。
- 结算后立即进行红线检测：任一属性 `<= redline` 则立刻进入对应崩解结局，run 终止。

### 数值显示规则

- 周事件(1..12周)阶段：不展示具体数值与数值变化。
- 仅通过反馈系统表现：
  - 正向：边缘淡绿微光
  - 负向：边缘暗红微光
  - 危险：当属性 `<= redline + 5` 时，对应方向边缘出现稳定红色微光(直到恢复安全)
- 最终报告：展示完整数值与雷达图。

## 红线与预警

- 红线表：见 `content/v3.3.0/rules.json`
- 预警阈值：`warn_threshold = redline + 5`

## 随机系统 (确定性)

所有随机行为必须仅依赖 `(run_seed, ruleset_version, week, choice_id, roll_index)` 等稳定输入，禁止使用系统时间。

### 每周“3选项展示”抽取

- 输入：`run_seed`、`week`、历史选择序列哈希(可选)
- 输出：该周展示的 3 个 option_id
- 要求：在相同输入下输出一致；并在服务端持久化“当周展示的 3 个选项”，用于续玩与防篡改验证。

### `±n` 变化 (整数，中心偏置)

当选项存在 `±n` (n 为整数)：

- 取 `k ~ Binomial(2n, 0.5)` (确定性伪随机生成)
- 令 `delta = k - n`
- delta 范围：`[-n, n]`，整数，分布中心偏置。

备注：实现上可以通过对 `2n` 次确定性 coin flip 求和得到 `k`。

## 属性取值范围与钳制(clamp)

服务端每次结算属性变化后进行钳制：

- clamp 到 `[0, 100]` (含)

说明：
- 该钳制用于数据一致性与防异常扩散；红线判断依旧在“结算后”进行，钳制并不会改变崩解是否发生(若降到负值也会被 clamp 到 0，仍然 <= 红线)。

## 决赛(第12周)战术锁与胜负

战术定义与属性要求来自 `content/v3.3.0/finals.json`。

### 属性达标

- 单属性要求：`attr >= req`
- 双属性要求(直觉反应)：`skill >= 60 && mind >= 60`

未达标：直接失败分支，并应用“未达标失败”的属性变化。

达标：进入胜负判定，并应用“达标成功”的属性变化，然后继续进入总结与报告。

### 胜率模型

达标时：

- `base_win_rate = 0.70`
- `excess` 计算：
  - 单属性：`excess = attr - req`
  - 双属性：`excess = min(skill-60, mind-60)`
- `win_rate = clamp(base_win_rate + floor(excess / 10) * 0.05, 0.0, 0.95)`
- 以确定性 RNG 根据 `win_rate` 进行一次伯努利判定决定胜负。

说明：该模型与设计文档保持一致(“(达标属性-要求值)/10 × 5%”)，并将 excess 取整为 step，以保证确定性与可预测边界。

### 胜利档位(华丽/一般/险胜)：按 excess 分档

仅在“达标且判定胜利”时计算胜利档位：

- `excess >= 10` → 华丽胜利
- `excess >= 5` → 一般胜利
- 其他(0..4) → 险胜

## 积分与评级

- 基础分：六维加权求和(权重来自 `content/v3.3.0/rules.json`)
- 决赛因子：来自 `content/v3.3.0/rules.json`
- 崩解因子：固定为 `0.80` (确认)

### 失败档位(惜败/失利)：确定性规则

若进入决赛：

- 若“属性达标”且随机判定失败：记为 `惜败`，使用 `final_lose_close` 因子(0.97)。
- 若“属性未达标”直接失败：记为 `失利`，使用 `final_lose` 因子(0.94)。

Stage 1 不实现“未进决赛”状态；`no_final` 因子仅为未来扩展保留。

### 最终分随机微调(确定性)

最终分在乘以因子后进行一次确定性微调：

- 取 `magnitude_pct` ∈ {1, 2, 3}，确定性随机
- 取 `sign` ∈ {+1, -1}，确定性随机且等概率
- `micro_multiplier = 1 + sign * (magnitude_pct / 100)`
- `final_score = round_half_up(base_score * factor * micro_multiplier)`

说明：该实现满足“±(1% - 3%)”且完全可复现。

评级区间见 `content/v3.3.0/rules.json`。

## 舍入规则(rounding)

为避免语言/平台对 `round()` 的差异(如银行家舍入)，统一使用：

- `round_half_up(x)`：对 `x >= 0`，返回 `floor(x + 0.5)`。
